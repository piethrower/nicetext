Best results come from plain text by a single author, several paragraphs long, in the language and tone you want the cover stories to mimic.

Public-domain text sources to download from:

  • Project Gutenberg: plain-text classics, decades of public-domain literature.
  • Standard Ebooks: curated, cleaned-up versions of Gutenberg titles.
  • Wikisource: historical documents and shorter pieces.

If you download from Gutenberg, trimming the license preamble and trailer (the all-caps notices at the start and end) gives cleaner output. Otherwise the parser reads it all.

Any file works. The parser ignores binary noise and short stray strings, so PDFs, eBooks, even executables can be uploaded. Text-heavy files (.txt, .md, scraped HTML, .epub stripped of markup) produce the cleanest sentence patterns. Larger inputs give richer sentence-shape variety.

Sentence order:
Whole-model output keeps the original sentence sequence. Split-model output mixes and matches sentence shapes.
Vocabulary:
Limit word choices to those in the sample text, or expand the vocabulary to include sources you select in the Codebook below. "Mary had a little lamb" might become "Mary had a white snow" (limited) or "Xavier owned a parsnickety armadillo" (expanded).
Codebook
No word-list loaded.

Custom is most useful for vocabulary the built-ins don't cover: industry jargon, character names from a novel, niche technical terms, regional slang. The eight built-in sources already cover general English; a small hand-curated list usually beats a larger generic one for stylistic flavor.

The text area uses one type,word pair per line. Paste, Load, and the editor accept tab- or space-delimited input too; the verify pass rewrites every valid line to the canonical comma form.

After paste, file load, or when you tab out of the text area, NiceText validates each line. Bad lines (whitespace inside the type or word, or words the lexer wouldn't accept) are kept in place but prefixed with # so the rest of the build still proceeds. Edit the offending line and tab out again to re-validate.

Lines starting with # are treated as comments. The same word may appear with multiple types; types merge naturally during build.
