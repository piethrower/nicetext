Sample texts (corpora)

All from Project Gutenberg, public domain:

  • JFK's Inaugural Address (US gov't work, 1961, eBook 3).
  • Aesop's Fables, translated by George Fyler Townsend (1887).
  • The Wonderful Wizard of Oz by L. Frank Baum (1900).
  • The Complete Works of William Shakespeare (Project Gutenberg eBook 100, current public-domain edition).
  • A Book of Nonsense by Edward Lear (Project Gutenberg eBook 982). Source for the Limerick card.
  • Shakespeare's Sonnets by William Shakespeare (Project Gutenberg eBook 1041). Standalone sonnet sequence, source for the Sonnet card.
  • Mother Goose's Nursery Rhymes (Project Gutenberg eBook 39784). Source for the Nursery Rhyme card.
  • The Complete Plays of Gilbert and Sullivan (Project Gutenberg eBook 808). Single-volume libretti for H.M.S. Pinafore, The Mikado, The Pirates of Penzance, and the rest of the Savoy operas. Source for the Gilbert & Sullivan card.
  • Japanese Literature, edited by Epiphanius Wilson, A.M. (1900; Project Gutenberg eBook 19264). Classical Japanese verse (waka / tanka, 5-7-5-7-7) alongside selections from Genji Monogatari. Source for the Tanka card. Note: PG's metadata attributes this volume to Basil Hall Chamberlain because of his translation contributions; the editorial frame is Wilson's.

All corpora are curated subsets of the original Project Gutenberg eBooks: narrative or verse body is retained, while front-matter (title pages, contents lists, prefaces, dedications, illustration blocks) and back-matter (indexes, glossaries, transcriber's notes, footnotes) are removed so that the corpus-trained sentence model captures the body of each work rather than ambient prose. Each file carries a CURATION NOTE block immediately above its *** START OF THE PROJECT GUTENBERG EBOOK ... *** marker describing exactly what was removed and linking to the complete original eBook on gutenberg.org. The Project Gutenberg preamble and license text remain intact in the file header and footer. The lexer strips both the PG preamble and the curation note at corpus-load time via its GUTENBERG_START / GUTENBERG_END tokens, so neither appears in generated cover text.

Word and type/word lists

  • MIT names and places list (mitlist/): Bob Baldwin's collection from MIT, augmented by Matt Bishop and Daniel Klein. Distributed under informal academic terms.
  • Morphological part-of-speech tagging (kimmo.twlist): output of PC-KIMMO with Englex (Antworth, SIL, 1995).
  • Synonym groupings (f2p.twlist): derived from the Frog2Prince thesis, UW–Milwaukee.
  • Carnegie Mellon Pronouncing Dictionary (fixture-src/pron/cmu/cmudict.dict): ARPABET pronunciations for ~135,000 English words, maintained by the Speech Group at cmusphinx/cmudict. © 1993–2015 Carnegie Mellon University; redistributed under the BSD 2-clause license shipped alongside the dictionary in fixture-src/pron/cmu/LICENSE. Source for every pronunciation-derived twlist (rhyme groups, syllable count, stress pattern, alliteration) and for the xanax rewriter's a/an agreement exception list.
  • Rhyming groups (rhyme.twlist): derived from the CMU Pronouncing Dictionary above.
  • Syllable-count buckets (cmu-syllable.twlist): derived from CMU by counting vowel phonemes per pronunciation. One type per syllable count (syl_1, syl_2, ...). Build: node fixture-src/twlist/cmu-syllable/derive.js.
  • Stress patterns (cmu-stress.twlist): derived from CMU by concatenating ARPABET stress digits (0/1/2) across the vowels of each pronunciation. One type per distinct sequence (stress_01 = iamb, stress_10 = trochee, stress_001 = anapest, stress_100 = dactyl). Build: node fixture-src/twlist/cmu-stress/derive.js.
  • Alliteration groups (cmu-alliteration.twlist): derived from CMU by taking each word's first phoneme. One type per phoneme (allit_K, allit_S, allit_TH, ...). Build: node fixture-src/twlist/cmu-alliteration/derive.js.
  • Numeric word lists: trivially generated, not copyrightable.
  • Modern words supplement (claude2026/): Claude-authored 2026 vocabulary additions covering actions, collectives, emotions, modifiers, nature, new words, objects, people, semantic-domain, sensory, and time-space categories.
  • Connector words (cfg-words/): per-grammar singleton entries used by CFG drop-rule semantics.
  • Typo / canonical pairs (fixture-src/rewriters/typos/): English (typo, canonical) substitution pairs derived from client9/misspell's DictMain word list. Copyright © 2015-2017 Nick Galbreath; MIT license. Source for the typos cover-transform rewriter (forward / reverse modes).
  • British / American spelling pairs (fixture-src/rewriters/british/): (source, target, direction) rows derived from client9/misspell's DictAmerican + DictBritish word lists. Copyright © 2015-2017 Nick Galbreath; MIT license. Source for the british cover-transform rewriter (us-uk Britishize / uk-us Americanize modes).
  • Voice opener / closer sample data (fixture-src/reformatters/voice/): hand-authored sample twlists for the voice reformatter (pirate mode in the MVP). Released as CC0 / public domain. Source for the voice cover-transform reformatter.

Programming-language keywords (proglang-keywords/)

Reserved keywords, language constants, and built-in identifiers across twelve programming languages, three shell dialects, and five command sets. The single-word identifiers in question (if, for, true, printf, grep, ...) are not copyrightable individually; the sources below are the public references each per-language word list was authored against.

  • C (c.twlist): ISO/IEC 9899 keyword list (§6.4.1) and the standard library headers (<stdio.h>, <stdlib.h>, <string.h>, etc.).
  • C++ (c++.twlist): ISO/IEC 14882 keyword list ([lex.key]) and the std namespace fundamentals.
  • COBOL (cobol.twlist): ISO/IEC 1989 reserved-word list (COBOL-85 / -2002 / -2014).
  • CSS (css.twlist): CSS Working Group drafts (drafts.csswg.org) cross-referenced with MDN Web Docs.
  • Fortran (fortran.twlist): ISO/IEC 1539 (Fortran 77 / 90 / 95 / 2003 / 2008 / 2018).
  • HTML (html.twlist): WHATWG HTML Living Standard (html.spec.whatwg.org) cross-referenced with the MDN HTML Elements reference.
  • Java (java.twlist): Java Language Specification (docs.oracle.com/javase/specs/) plus the java.lang package roster.
  • JavaScript (js.twlist): ECMAScript Language Specification (tc39.es/ecma262/) and the universally-available global builtins documented by MDN.
  • Perl (perl.twlist): Perl 5 core documentation (perldoc.perl.org): perlfunc, perlvar, perlop.
  • PHP (php.twlist): PHP manual (reserved.keywords and core predefined functions).
  • Python (python.twlist): CPython lexical analysis, built-in functions, and the data model (dunder protocol).
  • SQL (sql.twlist): ISO/IEC 9075 reserved words plus the Wikipedia cross-vendor reserved-word list (errs toward the union).
  • POSIX sh (sh.twlist): IEEE Std 1003.1 Shell Command Language + POSIX special built-in utilities.
  • Bash (bash.twlist): GNU Bash Reference Manual (4.x / 5.x).
  • csh / tcsh (csh.twlist): the tcsh(1) manual page and the BSD csh(1) manual.
  • Linux commands (linux-commands.twlist): GNU coreutils, util-linux, systemd, iproute2 / net-tools manifests, plus the command names shipped by major distro package managers (apt / dnf / yum / rpm / pacman / zypper / snap / flatpak).
  • BSD commands (bsd-commands.twlist): FreeBSD man-page section 1/8 and the OpenBSD base userland set.
  • DOS commands (dos-commands.twlist): MS-DOS 6.22 command reference and the COMMAND.COM internal-command list, plus standard DOS external utilities.
  • Windows commands (windows-commands.twlist): the Microsoft Learn Windows Commands reference (learn.microsoft.com).
  • PowerShell cmdlets (powershell-commands.twlist): the Microsoft Learn PowerShell documentation (learn.microsoft.com/powershell/) and the Microsoft.PowerShell.* core module manifests.

Unicode® Emoji 16.0 + CLDR 48

Two Unicode Consortium data sources drive the emoji TW-list. Both are © Unicode, Inc.; redistribution is permitted under the Unicode Terms of Use (https://www.unicode.org/copyright.html):

  • Emoji (Unicode 16) (fixtures/emoji16.twlist.tsv.gz): derived from emoji-test.txt at unicode.org/Public/emoji/16.0/. Each fully-qualified entry (3,766 admitted of 3,784, including all skin-tone and gender variants) ships as a separate bit-bearing row. Subgroup headers are carried verbatim, snake-cased, as em16_<subgroup> types.
  • Emoji-related Phrases (CLDR 48) (fixtures/emoji-cldr-names-16.twlist.tsv.gz): multi-word emoji names (the CLDR tts spoken-name field, e.g. "alarm clock", "United States" with the "flag: " prefix stripped from country flags), typed under each emoji's home Unicode subgroup so cross-modal slots compose at runtime. Source: CLDR 48 English annotations and annotationsDerived.
  • CLDR keyword sidecar (fixtures/emoji16.cldr.json.gz): per-emoji English keyword list ("rose | red | flower | …") used by the cross-modal augmentation passes (Aug A / Aug B / Aug-mix). Same source files as Emoji-related Phrases, but the tts name field is split out into the phrase fixture above and the pipe-separated keyword arrays land here as JSON.
  • Emoji Phrases (hand-curated) (fixtures/emoji-curated-phrases-16.twlist.tsv.gz): hand-curated multi-emoji and mixed Latin+emoji phrases (e.g. 🌹 💐, happy 😀, hello 👋) typed under em16_<subgroup> labels so they compete for cover slots with single-emoji entries from the Emoji source. Phrase content is original to this project; the underlying emoji and subgroup labels derive from Unicode Emoji 16.0 above.
  • Emoji Curated Keywords (CLDR 48) (fixtures/emoji16.curated-keywords.tsv.gz): AI-curated subset of the CLDR keyword set (3,573 entries from 3,689). When ticked as a BYOS source, the augmentation passes filter their CLDR keyword input to this list (default: full CLDR keyword set, mechanical / noisier). Curation drops non-Latin / multi-word / numeric / symbolic strings plus a small English function-word stoplist whose narrow- phrase emissions ("the 🌹", "for 🌹") read as stilted. Curation method documented inline at fixture-src/twlist/emoji16/curated-keywords.txt.

Unicode® TR39 Confusables 15.1.0

The build-time corpus pre-clean module folds visually-confusable non-Latin code points (Cyrillic а, Greek α, Cherokee letters that mimic Latin caps, &c.) to their Latin look-alikes so words like pаper don't fragment at the script boundary inside the lexer. The fold table is derived from Unicode TR39 confusables.txt v15.1.0, fetched into fixture-src/confusables/raw/ and filtered offline into fixture-src/confusables/cooked/confusables-data.js via tools/build-confusables-map.js (deployed to fixtures/confusables-data.js). We redistribute only the filtered fold table, not the upstream file.

Random-secret format references

NiceText's "Make a random secret" feature offers two output shapes designed to be visually indistinguishable from the genuine output of two popular file-encryption tools, so an interceptor who recovers the smuggled payload cannot trivially distinguish it from real ciphertext from those tools. Each shape is derived solely from the tool's publicly documented file format. No code, binaries, libraries, headers, or other intellectual property from either project is included, linked to, or invoked at any point. NiceText is an independent project and is not endorsed by, affiliated with, or sponsored by either organization. The names and marks below are used only to accurately describe which file shape is being mimicked.

  • OpenSSL enc shape: 8-byte ASCII Salted__ magic + 8 random salt bytes + AES-CBC body (16-byte aligned), matching the salted output of the openssl(1) enc command. No OpenSSL code runs and no AES encryption is performed; the body bytes are cryptographically random. OpenSSL is a trademark of the OpenSSL Software Foundation. See openssl.org for the project, documentation, and trademark policy.
  • VeraCrypt container shape: pure random bytes, sector-aligned to 512 bytes, with a 292 KB minimum, matching the size and layout of a VeraCrypt file container. A real VeraCrypt container's header lives encrypted inside the first 512 bytes; we do not synthesize that header (none is required for the file to be visually indistinguishable from random data, and synthesizing one would require touching VeraCrypt's cryptographic specification rather than just its shape). No VeraCrypt code runs. VeraCrypt is a trademark of its respective owners. See veracrypt.io for the project and documentation.

Code adapted from sibling projects

NiceText shares conventions with a sibling personal project, Span-It! (planned for release at piethrower.github.io/spanit). Two specific pieces of NiceText's code are adapted from Span-It! rather than written fresh:

  • Matrix digital rain (js/matrix-rain.js): canvas-based effect adapted from Span-It!'s file of the same name, with differences in DOM-observation, pause/resume, and z-index handling noted in the file header.
  • Penny tutorial guide (js/penny.js, css/penny.css, img/penny.svg): tutorial character pattern, typewriter reveal, and snap-to-line bubble sizing ported in spirit from Span-It!'s Penny, then rewritten as a smaller spec for NiceText.

The repo layout convention (HTML pages at the root, sibling css/ / js/ / img/ dirs, tests under tests/node/) also follows Span-It!.
